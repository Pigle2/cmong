# 사용자 여정 맵

## 1. 구매자(의뢰인) 여정

### 1.1 전체 여정 개요

```
[인지] → [탐색] → [비교/검토] → [결제] → [작업 진행] → [검수/완료] → [재구매]
```

### 1.2 단계별 상세 여정

#### Stage 1: 인지 (Awareness)

| 항목 | 내용 |
|------|------|
| **상황** | 전문 서비스가 필요하다는 것을 인식 |
| **접점** | 검색엔진, SNS 광고, 지인 추천, 블로그/커뮤니티 |
| **행동** | 프리랜서 플랫폼 검색, 서비스 후기 탐색 |
| **생각** | "어디서 맡기면 좋을까?", "품질이 괜찮을까?" |
| **감정** | 기대 + 불안 |
| **기회** | SEO 최적화, 후기 기반 마케팅, 첫 주문 할인 쿠폰 |

#### Stage 2: 탐색 (Exploration)

| 항목 | 내용 |
|------|------|
| **상황** | 플랫폼에 방문하여 서비스를 탐색 |
| **접점** | 메인페이지, 카테고리, 검색, 추천 서비스 |
| **행동** | 카테고리 브라우징, 키워드 검색, 필터링 |
| **생각** | "내가 필요한 서비스가 있을까?", "어떤 기준으로 고를까?" |
| **감정** | 호기심 |
| **기회** | AI 추천, 인기 서비스 큐레이션, 직관적 검색 UX |

#### Stage 3: 비교/검토 (Consideration)

| 항목 | 내용 |
|------|------|
| **상황** | 여러 전문가의 서비스를 비교 |
| **접점** | 서비스 상세 페이지, 포트폴리오, 리뷰 |
| **행동** | 포트폴리오 확인, 리뷰 읽기, 가격 비교, 판매자에게 문의 |
| **생각** | "이 사람 실력이 괜찮은가?", "가격이 합리적인가?" |
| **감정** | 신중함, 비교 고민 |
| **기회** | 상세한 포트폴리오, 구매자 리뷰, 판매자 응답률/속도 표시 |

#### Stage 4: 결제 (Purchase)

| 항목 | 내용 |
|------|------|
| **상황** | 서비스를 선택하고 결제 진행 |
| **접점** | 주문서 작성, 결제 페이지 |
| **행동** | 옵션 선택, 요구사항 작성, 결제 수단 선택, 결제 완료 |
| **생각** | "에스크로니까 안전하겠지", "요구사항을 명확히 적어야지" |
| **감정** | 기대 + 약간의 긴장 |
| **기회** | 에스크로 안내, 요구사항 템플릿 제공, 간편결제 |

#### Stage 5: 작업 진행 (In Progress)

| 항목 | 내용 |
|------|------|
| **상황** | 전문가가 작업을 진행하는 기간 |
| **접점** | 채팅, 작업 진행률 대시보드, 알림 |
| **행동** | 진행 상황 확인, 중간 산출물 검토, 피드백 전달 |
| **생각** | "제대로 진행되고 있나?", "일정은 맞출까?" |
| **감정** | 기다림, 불안 (진행 상황 미공유 시) |
| **기회** | 자동 진행 상황 알림, 마일스톤 트래킹, 채팅 내 파일 공유 |

#### Stage 6: 검수/완료 (Review & Complete)

| 항목 | 내용 |
|------|------|
| **상황** | 최종 결과물을 받고 검수 |
| **접점** | 납품 알림, 결과물 확인, 수정 요청, 구매 확정 |
| **행동** | 결과물 확인, 수정 요청 또는 구매 확정, 리뷰 작성 |
| **생각** | "결과물이 만족스러운가?", "수정이 필요한가?" |
| **감정** | 만족 또는 실망 |
| **기회** | 명확한 수정 정책 안내, 리뷰 작성 유도, 재주문 할인 |

#### Stage 7: 재구매 (Retention)

| 항목 | 내용 |
|------|------|
| **상황** | 만족한 경우 재구매 고려 |
| **접점** | 마이페이지, 이메일/푸시 알림, 찜 목록 |
| **행동** | 같은 전문가에게 재주문, 다른 서비스 탐색 |
| **생각** | "이전에 만족했으니 또 맡기자" |
| **감정** | 신뢰, 편안함 |
| **기회** | 재주문 할인, 전문가 팔로우, 맞춤 추천 |

## 2. 판매자(전문가) 여정

### 2.1 전체 여정 개요

```
[가입] → [프로필 설정] → [서비스 등록] → [의뢰 수주] → [작업 수행] → [납품/정산] → [성장]
```

### 2.2 단계별 상세 여정

#### Stage 1: 가입 및 프로필 설정

| 항목 | 내용 |
|------|------|
| **상황** | 프리랜서로서 수익 창출 채널을 찾고 있음 |
| **접점** | 회원가입, 판매자 전환, 프로필 설정 |
| **행동** | 회원가입, 실명 인증, 전문 분야 선택, 포트폴리오 업로드 |
| **생각** | "이 플랫폼에서 수익을 낼 수 있을까?" |
| **감정** | 기대 + 불확실 |
| **기회** | 간편한 가입 프로세스, 프로필 완성도 가이드, 성공 사례 공유 |

#### Stage 2: 서비스(긱) 등록

| 항목 | 내용 |
|------|------|
| **상황** | 판매할 서비스를 등록 |
| **접점** | 서비스 등록 폼, 가격 설정, 포트폴리오 첨부 |
| **행동** | 서비스 제목/설명 작성, 가격/옵션 설정, 포트폴리오 업로드 |
| **생각** | "어떻게 해야 잘 팔릴까?", "가격은 얼마가 적당할까?" |
| **감정** | 고민, 기대 |
| **기회** | 서비스 등록 가이드, 인기 키워드 추천, 적정 가격 가이드 |

#### Stage 3: 의뢰 수주

| 항목 | 내용 |
|------|------|
| **상황** | 구매자로부터 주문 또는 문의를 받음 |
| **접점** | 알림(푸시/이메일), 주문 관리, 채팅 |
| **행동** | 주문 확인, 구매자 요구사항 확인, 작업 일정 협의 |
| **생각** | "이 프로젝트를 맡을 수 있을까?", "일정이 맞을까?" |
| **감정** | 기대, 부담 |
| **기회** | 빠른 알림, 주문 수락/거절 기능, 자동 일정 관리 |

#### Stage 4: 작업 수행

| 항목 | 내용 |
|------|------|
| **상황** | 의뢰받은 작업을 수행 |
| **접점** | 채팅, 파일 공유, 진행률 업데이트 |
| **행동** | 작업 진행, 중간 산출물 공유, 피드백 반영 |
| **생각** | "구매자가 만족할까?", "수정이 많지 않았으면" |
| **감정** | 집중, 긴장 |
| **기회** | 마일스톤 관리 도구, 수정 횟수 관리, 작업 템플릿 |

#### Stage 5: 납품 및 정산

| 항목 | 내용 |
|------|------|
| **상황** | 작업 완료 후 결과물 납품 및 대금 정산 |
| **접점** | 납품 기능, 정산 관리, 리뷰 확인 |
| **행동** | 최종 결과물 납품, 구매 확정 대기, 정산 확인 |
| **생각** | "빨리 구매 확정됐으면", "정산은 언제 되지?" |
| **감정** | 안도, 기대 |
| **기회** | 자동 구매 확정, 빠른 정산 옵션, 리뷰 알림 |

#### Stage 6: 성장

| 항목 | 내용 |
|------|------|
| **상황** | 실적을 쌓으며 등급과 수익 성장 |
| **접점** | 대시보드, 통계, 등급 시스템 |
| **행동** | 매출 분석, 서비스 최적화, 신규 서비스 등록, 등급 확인 |
| **생각** | "어떻게 하면 더 많은 주문을 받을까?" |
| **감정** | 성취감, 도전 |
| **기회** | 판매 팁 제공, 성공 사례 공유, 등급 달성 보상 |

## 3. 핵심 터치포인트 및 감정 곡선

### 3.1 구매자 감정 곡선

```
감정
 ↑
 5 │          ★                              ★★★
   │         ╱ ╲                            ╱
 4 │        ╱   ╲         ★               ╱
   │       ╱     ╲       ╱ ╲             ╱
 3 │  ★   ╱       ╲     ╱   ╲           ╱
   │   ╲ ╱         ╲   ╱     ╲         ╱
 2 │    ★            ╲ ╱       ╲       ╱
   │                   ★        ╲     ╱
 1 │                              ★★★
   │
   └──────────────────────────────────────▶ 시간
     방문  가입  검색  상세  결제  대기  검수  확정
```

| 구간 | 감정 수준 | 설명 |
|------|-----------|------|
| 방문 | 3 (보통) | 기대와 불안이 공존 |
| 가입 | 2 (낮음) | 가입 절차가 번거로우면 이탈 가능 |
| 검색/탐색 | 5 (높음) | 다양한 서비스 발견 시 흥미 |
| 서비스 상세 | 4 (높음) | 포트폴리오/리뷰로 신뢰 형성 |
| 결제 | 2 (낮음) | 결제 불안감 (에스크로 안내 필요) |
| 대기 (작업 진행) | 1~4 (변동) | 진행 상황 공유 여부에 따라 크게 변동 |
| 검수 | 1~5 (변동) | 결과물 만족도에 따라 극명하게 갈림 |
| 구매 확정 | 5 (높음) | 만족스러운 결과물 수령 시 최고점 |

### 3.2 핵심 개선 포인트

| # | 터치포인트 | 문제 | 개선 방안 |
|---|-----------|------|-----------|
| 1 | 가입 | 복잡한 절차 | 소셜 로그인, 최소 정보만 수집 |
| 2 | 검색 | 원하는 서비스를 못 찾음 | AI 추천, 스마트 검색, 필터 고도화 |
| 3 | 결제 | 결제 불안 | 에스크로 보호 강조, 100% 환불 보장 안내 |
| 4 | 작업 대기 | 진행 상황 불투명 | 자동 진행률 알림, 마일스톤 트래킹 |
| 5 | 검수 | 기대와 다른 결과물 | 중간 검수, 수정 정책 명확화 |
| 6 | 정산(판매자) | 정산 지연 | 빠른 정산 옵션, 투명한 정산 현황 |
