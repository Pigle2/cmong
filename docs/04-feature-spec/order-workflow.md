# 주문/거래 프로세스 기능 명세

## 1. 기능 개요

| 항목 | 내용 |
|------|------|
| **기능명** | 주문/거래 프로세스 |
| **목적** | 구매자와 판매자 간 서비스 거래의 전체 워크플로우 관리 |
| **사용자** | 구매자, 판매자 |

### 2.2 주문 상태

| 상태 | 코드 | 설명 |
|------|------|------|
| **결제 완료** | `PAID` | 구매자 결제 완료, 판매자 수락 대기 |
| **주문 수락** | `ACCEPTED` | 판매자 주문 수락, 작업 시작 |
| **주문 거절** | `REJECTED` | 판매자 주문 거절, 전액 환불 |
| **작업중** | `IN_PROGRESS` | 판매자 작업 진행 중 |
| **납품 완료** | `DELIVERED` | 판매자 결과물 납품, 구매자 검수 대기 |
| **수정 요청** | `REVISION_REQUESTED` | 구매자 수정 요청, 판매자 재작업 |
| **구매 확정** | `COMPLETED` | 구매자 구매 확정 또는 자동 확정 |
| **취소** | `CANCELLED` | 주문 취소 (결제 전/환불) |
| **분쟁중** | `DISPUTED` | 구매자/판매자 분쟁 발생, 운영팀 중재 |
| **환불 완료** | `REFUNDED` | 환불 처리 완료 |

## 3. 주문서 작성

### 3.1 주문서 입력 필드

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| 패키지 선택 | Radio | O | Standard / Deluxe / Premium |
| 추가 옵션 | Checkbox | X | 판매자가 설정한 추가 옵션 |
| 요구사항 | Textarea | O | 구매자의 상세 요구사항 (최소 30자) |
| 참고 파일 | File | X | 참고 자료 첨부 (최대 5개, 각 50MB) |
| 희망 납기일 | Date | X | 기본값: 패키지 작업일 기준 |
| 쿠폰 적용 | Select | X | 보유 쿠폰 목록에서 선택 |

### 3.2 주문 금액 계산

```
서비스 금액    = 패키지 가격 + 추가 옵션 합계
쿠폰 할인      = 쿠폰 유형별 할인 적용
포인트 사용    = 최대 결제금액의 50%까지
결제 수수료    = (서비스 금액 - 쿠폰 할인 - 포인트 사용) × 결제수단별 수수료율
──────────────────────────────────────────
총 결제 금액   = 서비스 금액 - 쿠폰 할인 - 포인트 사용 + 결제 수수료

※ 수수료율은 결제수단에 따라 상이 (신용카드 3.5%, 간편결제 3.3%, 계좌이체 1.5%, 가상계좌 건당 300원)
```

## 4. 작업 진행 관리

### 4.1 마일스톤 관리

| 마일스톤 | 설명 | 자동/수동 |
|----------|------|-----------|
| 요구사항 확인 | 구매자 요구사항을 판매자가 확인 | 수동 |
| 중간 산출물 | 중간 결과물 공유 (선택) | 수동 |
| 최종 납품 | 최종 결과물 납품 | 수동 |
| 검수 완료 | 구매자 검수 확인 | 수동 |

### 4.2 수정 요청

| 항목 | 규칙 |
|------|------|
| 수정 가능 횟수 | 패키지별 설정된 횟수 이내 |
| 추가 수정 | 수정 횟수 초과 시 추가 수정 옵션 구매 가능 |
| 수정 요청 방법 | 채팅 또는 수정 요청 폼 작성 |
| 수정 기한 | 수정 요청 후 기존 작업일의 50% 이내 |

### 4.3 납기 관리

| 상황 | 처리 |
|------|------|
| 정상 납품 | 납기일 이내 납품 완료 |
| 납기 지연 | 판매자에게 경고 알림, 구매자에게 안내 |
| 납기 초과 (3일) | 구매자 무조건 취소 가능 (전액 환불) |
| 납기 연장 요청 | 판매자가 사유와 함께 연장 요청 → 구매자 승인 필요 |

## 5. 구매 확정

### 5.1 확정 프로세스

| 방법 | 설명 |
|------|------|
| **수동 확정** | 구매자가 결과물 검수 후 "구매 확정" 클릭 |
| **자동 확정** | 납품 완료 후 5일 경과 시 자동 확정 (이의 신청 없을 경우) |

### 5.2 확정 후 처리

```
구매 확정
├── 판매자 수익 확정 (수수료 차감 후)
├── 정산 대기열에 추가
├── 구매자에게 리뷰 작성 요청 알림
└── 거래 완료 처리
```

## 6. 취소/환불

### 6.1 취소 정책

| 시점 | 구매자 취소 | 판매자 취소 |
|------|------------|------------|
| 주문 수락 전 | 전액 환불 | 전액 환불 |
| 수락 후 ~ 작업 시작 전 | 전액 환불 | 전액 환불 (패널티 부과) |
| 작업 시작 후 ~ 중간 납품 전 | 50% 환불 (협의 가능) | 전액 환불 (패널티 부과) |
| 중간 납품 이후 | 환불 불가 (분쟁 조정 가능) | 불가 |
| 납기 초과 (3일) | 구매자 무조건 취소 가능 (전액 환불) | - |

### 6.2 판매자 귀책 패널티

| 패널티 | 조건 |
|--------|------|
| 주문 수락률 하락 | 거절/취소 1건당 수락률 감소 |
| 경고 1회 | 판매자 귀책 취소 1건 |
| 서비스 노출 제한 (7일) | 경고 누적 3회 |
| 판매자 자격 정지 (30일) | 경고 누적 5회 |

## 7. 분쟁 처리

### 7.1 분쟁 프로세스

```
1. 구매자 이의 신청 (납품 후 5일 이내)
   └── 사유 선택 + 상세 설명 작성 + 증빙 자료 첨부
2. 판매자 소명 (3일 이내)
   └── 반박 의견 + 증빙 자료 첨부
3. 운영팀 검토 (7일 이내)
   └── 양측 자료 검토, 필요 시 추가 자료 요청
4. 결정 및 처리
   ├── 구매자 유리: 전액/부분 환불
   ├── 판매자 유리: 거래 확정
   └── 합의 유도: 부분 환불 + 추가 작업 등
```

### 7.2 분쟁 사유 분류

| 사유 | 설명 |
|------|------|
| 결과물 품질 불만 | 기대 수준 미달 |
| 요구사항 미반영 | 주문 시 요구한 내용 누락 |
| 납기 초과 | 약속한 작업일 초과 |
| 의사소통 불가 | 판매자 연락 두절 |
| 기타 | 기타 사유 |

## 8. 비즈니스 룰

| # | 규칙 |
|---|------|
| BR-ORD-01 | 주문 수락은 결제 후 24시간 이내 처리 필요 (미처리 시 자동 취소) |
| BR-ORD-02 | 수정 요청은 패키지별 설정 횟수 이내만 무료 |
| BR-ORD-03 | 자동 구매 확정은 납품 후 5일 경과 + 이의 신청 없음 조건 |
| BR-ORD-04 | 판매자 주문 수락률이 70% 미만이면 검색 노출 페널티 |
| BR-ORD-05 | 구매자가 24시간 이내 요구사항 미제출 시 알림 발송 |
| BR-ORD-06 | 분쟁 중인 주문은 정산 보류 |
| BR-ORD-07 | 동일 판매자에 대한 동시 진행 주문 최대 3건 |
