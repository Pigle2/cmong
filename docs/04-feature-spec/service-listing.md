# 서비스(긱) 등록/관리 기능 명세

## 1. 기능 개요

| 항목 | 내용 |
|------|------|
| **기능명** | 서비스(긱) 등록 및 관리 |
| **목적** | 판매자가 자신의 전문 서비스를 등록하고 관리 |
| **사용자** | 판매자 |

---

## 2. 서비스 등록

### 2.1 등록 단계

```
Step 1: 카테고리 선택
Step 2: 기본 정보 입력
Step 3: 가격/패키지 설정
Step 4: 상세 설명 및 포트폴리오
Step 5: FAQ 및 추가 설정
Step 6: 미리보기 및 등록 요청
```

### 2.2 입력 필드 상세

#### Step 1: 카테고리 선택

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| 대분류 | Select | O | 10개 대분류 중 선택 |
| 중분류 | Select | O | 대분류에 따른 중분류 |
| 소분류 | Select | O | 중분류에 따른 소분류 |
| 서비스 태그 | Tag Input | X | 검색용 태그 (최대 10개) |

#### Step 2: 기본 정보

| 필드 | 타입 | 필수 | 규칙 | 설명 |
|------|------|:----:|------|------|
| 서비스 제목 | Text | O | 10~50자 | 서비스를 한 줄로 설명 |
| 대표 이미지 | Image | O | JPG/PNG, 최대 5MB, 권장 750x500px | 검색/목록에 표시되는 대표 이미지 |
| 추가 이미지 | Image | X | 최대 5장 | 서비스를 설명하는 추가 이미지 |
| 서비스 소개 (짧은) | Textarea | O | 50~200자 | 검색 결과에 표시되는 짧은 소개 |

#### Step 3: 가격/패키지 설정

**패키지 타입**: 3단계 패키지 (Standard / Deluxe / Premium)

| 필드 | 타입 | 필수 | 규칙 |
|------|------|:----:|------|
| 패키지명 | Text | O | 각 패키지별 이름 |
| 가격 | Number | O | 최소 5,000원, 최대 10,000,000원 |
| 작업일 | Number | O | 1~90일 |
| 수정 횟수 | Number | O | 0~무제한 |
| 패키지 설명 | Textarea | O | 패키지에 포함된 내용 |
| 추가 옵션 | List | X | 옵션명 + 추가 금액 + 추가 작업일 |

**가격 설정 예시**:

| 패키지 | 가격 | 작업일 | 수정 | 포함 내용 |
|--------|------|--------|------|-----------|
| Standard | 50,000원 | 3일 | 1회 | 로고 시안 1개 |
| Deluxe | 100,000원 | 5일 | 3회 | 로고 시안 3개 + 명함 디자인 |
| Premium | 200,000원 | 7일 | 5회 | 로고 시안 5개 + 명함 + 브랜드 가이드 |

**추가 옵션 예시**:

| 옵션명 | 추가 금액 | 추가 작업일 |
|--------|-----------|-------------|
| 시안 추가 1개 | +20,000원 | +1일 |
| 급행 작업 | +50,000원 | 0일 |
| 소스 파일 제공 | +30,000원 | 0일 |

#### Step 4: 상세 설명 및 포트폴리오

| 필드 | 타입 | 필수 | 규칙 |
|------|------|:----:|------|
| 상세 설명 | Rich Text | O | 200~5,000자, 이미지 삽입 가능 |
| 포트폴리오 | File/Image | O | 최소 1개, 최대 20개 |
| 작업 프로세스 | Textarea | X | 작업 진행 방식 안내 |

#### Step 5: FAQ 및 추가 설정

| 필드 | 타입 | 필수 | 규칙 |
|------|------|:----:|------|
| 자주 묻는 질문 | Q&A List | X | 최대 10개 |
| 주문 전 필수 확인사항 | Textarea | X | 구매자에게 사전 안내 |
| 주문 시 필요 정보 | Checklist | X | 구매자가 주문 시 제출해야 할 정보 |

## 3. 서비스 상태 관리

### 3.1 서비스 상태

```
[임시저장] → [심사대기] → [승인/반려]
                              │
                        ┌─────┼─────┐
                        ▼           ▼
                    [판매중]     [반려]
                        │         │
                    ┌───┴───┐     ▼
                    ▼       ▼  [수정 후 재심사]
                [판매중지] [삭제]
```

| 상태 | 설명 |
|------|------|
| **임시저장** | 작성 중인 서비스, 노출되지 않음 |
| **심사대기** | 등록 요청 후 관리자 심사 대기 |
| **판매중** | 심사 승인 후 정상 노출 |
| **반려** | 심사 거절, 반려 사유 안내 |
| **판매중지** | 판매자가 일시적으로 판매 중지 |
| **삭제** | 서비스 삭제 (진행 중인 주문이 없는 경우만 가능) |

### 3.2 심사 기준

| # | 기준 | 설명 |
|---|------|------|
| 1 | 카테고리 적합성 | 선택한 카테고리와 서비스 내용 일치 여부 |
| 2 | 서비스 설명 충실도 | 제목, 설명, 포트폴리오 품질 |
| 3 | 가격 적정성 | 시장 대비 합리적인 가격 설정 |
| 4 | 법적 문제 | 저작권 침해, 불법 서비스 여부 |
| 5 | 이미지 품질 | 대표 이미지/포트폴리오 해상도 및 적합성 |

## 4. 서비스 관리 기능

### 4.1 기능 목록

| 기능 | 설명 |
|------|------|
| **서비스 수정** | 등록 정보 수정 (가격 변경 시 진행 중 주문 미영향) |
| **서비스 복제** | 기존 서비스를 복제하여 새 서비스 생성 |
| **판매 중지/재개** | 일시적으로 서비스 노출 중지/재개 |
| **서비스 삭제** | 진행 중인 주문이 없는 경우 삭제 가능 |
| **통계 조회** | 조회수, 주문수, 전환율 등 확인 |
| **순서 변경** | 프로필 내 서비스 노출 순서 변경 |

### 4.2 서비스 등록 제한

| 판매자 등급 | 등록 가능 수 |
|------------|:----------:|
| 신규 | 5개 |
| 일반 | 15개 |
| 프로 | 무제한 |
| 마스터 | 무제한 |

## 5. 비즈니스 룰

| # | 규칙 |
|---|------|
| BR-SVC-01 | 서비스 등록 후 관리자 심사 필수 (최대 24시간 이내 심사 완료) |
| BR-SVC-02 | 가격 변경 시 기존 진행 중인 주문에는 영향 없음 |
| BR-SVC-03 | 서비스 삭제는 진행 중인 주문이 0건일 때만 가능 |
| BR-SVC-04 | 3회 이상 반려 시 해당 카테고리 등록 30일 제한 |
| BR-SVC-05 | 대표 이미지에 연락처(전화번호, 이메일 등) 포함 불가 |
| BR-SVC-06 | 타 플랫폼 유도 문구 포함 시 반려 |
| BR-SVC-07 | 서비스 수정 시 주요 변경사항(가격, 작업일)은 재심사 필요 |

## 6. 예외 처리

| 시나리오 | 처리 방안 |
|----------|-----------|
| 이미지 업로드 실패 | 자동 재시도 3회, 실패 시 "파일 크기나 형식을 확인해주세요" 안내 |
| 임시저장 데이터 손실 | 자동 저장(30초마다) + 수동 저장 버튼 |
| 심사 지연 | 24시간 초과 시 판매자에게 안내 알림, 자동 에스컬레이션 |
| 중복 서비스 감지 | 유사한 제목/설명의 기존 서비스 존재 시 경고 |
