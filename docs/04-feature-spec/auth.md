# 회원가입/로그인 기능 명세

## 1. 기능 개요

| 항목 | 내용 |
|------|------|
| **기능명** | 회원가입 및 로그인 |
| **목적** | 사용자 계정 생성, 인증, 세션 관리 |
| **사용자** | 구매자, 판매자, 관리자 |

---

## 2. 회원가입

### 2.1 가입 방식

| 방식 | 설명 |
|------|------|
| **이메일 가입** | 이메일 + 비밀번호 기반 가입 |
| **카카오 로그인** | 카카오 OAuth 2.0 연동 |
| **네이버 로그인** | 네이버 OAuth 2.0 연동 |
| **구글 로그인** | Google OAuth 2.0 연동 |
| **애플 로그인** | Apple Sign In 연동 (iOS 필수) |

### 2.2 이메일 가입 프로세스

```
1. 약관 동의
   ├── [필수] 서비스 이용약관
   ├── [필수] 개인정보 처리방침
   ├── [선택] 마케팅 수신 동의
   └── [선택] 이벤트/혜택 알림 수신
2. 이메일 입력 → 중복 확인 → 인증 메일 발송
3. 이메일 인증 코드 입력 (6자리, 유효시간 5분)
4. 비밀번호 설정
5. 닉네임 입력 → 중복 확인
6. 가입 유형 선택 (구매자/판매자)
7. 가입 완료
```

### 2.3 입력 필드 유효성 검증

| 필드 | 규칙 | 에러 메시지 |
|------|------|------------|
| **이메일** | 이메일 형식, 최대 100자 | "올바른 이메일 형식을 입력해주세요" |
| **비밀번호** | 8~20자, 영문+숫자+특수문자 조합 | "비밀번호는 8~20자, 영문/숫자/특수문자를 포함해야 합니다" |
| **비밀번호 확인** | 비밀번호와 일치 | "비밀번호가 일치하지 않습니다" |
| **닉네임** | 2~20자, 한글/영문/숫자, 특수문자 불가 | "닉네임은 2~20자, 한글/영문/숫자만 가능합니다" |
| **이메일 인증코드** | 숫자 6자리 | "인증코드가 올바르지 않습니다" |

### 2.4 소셜 가입 프로세스

```
1. 소셜 로그인 버튼 클릭
2. 소셜 서비스 인증 화면으로 이동
3. 인증 완료 → 콜백 처리
4. 기존 회원 여부 확인
   ├── 기존 회원: 자동 로그인
   └── 신규 회원: 추가 정보 입력
       ├── 약관 동의
       ├── 닉네임 설정 (소셜 닉네임 기본값)
       └── 가입 유형 선택
5. 가입 완료
```

### 2.5 비즈니스 룰

| # | 규칙 |
|---|------|
| BR-AUTH-01 | 동일 이메일로 중복 가입 불가 |
| BR-AUTH-02 | 소셜 계정과 동일 이메일의 일반 계정은 연동 가능 |
| BR-AUTH-03 | 탈퇴 후 30일 이내 동일 이메일 재가입 불가 |
| BR-AUTH-04 | 이메일 인증은 가입 후 24시간 내 완료 필수 |
| BR-AUTH-05 | 만 14세 미만 가입 불가 |

## 3. 로그인

### 3.1 로그인 방식

| 방식 | 설명 |
|------|------|
| 이메일 + 비밀번호 | 일반 로그인 |
| 소셜 로그인 | 카카오/네이버/구글/애플 |
| 자동 로그인 | Remember Me 토큰 기반 |

### 3.2 로그인 프로세스

```
1. 이메일/비밀번호 입력
2. 자동 로그인 체크 (선택)
3. 로그인 버튼 클릭
4. 서버 인증
   ├── 성공: JWT 토큰 발급 → 메인페이지 이동
   ├── 실패 (5회 미만): "이메일 또는 비밀번호가 올바르지 않습니다"
   └── 실패 (5회 이상): 계정 잠금 (30분) + CAPTCHA 적용
```

### 3.3 인증 토큰 정책

| 항목 | 값 |
|------|-----|
| **Access Token** | JWT, 유효기간 1시간 |
| **Refresh Token** | JWT, 유효기간 14일 (자동 로그인 시 30일) |
| **토큰 저장** | HttpOnly Secure Cookie (웹) / Secure Storage (앱) |
| **갱신** | Access Token 만료 시 Refresh Token으로 자동 갱신 |

### 3.4 보안 정책

| # | 정책 |
|---|------|
| SEC-01 | 비밀번호 5회 연속 실패 시 계정 잠금 (30분) |
| SEC-02 | 새로운 기기/브라우저 로그인 시 이메일 알림 |
| SEC-03 | 비밀번호는 bcrypt 해싱 후 저장 |
| SEC-04 | 모든 API 요청에 CSRF 토큰 적용 (웹) |
| SEC-05 | 동시 로그인 세션 최대 5개, 초과 시 가장 오래된 세션 종료 |

## 4. 비밀번호 찾기/재설정

### 4.1 프로세스

```
1. 가입 이메일 입력
2. 이메일 존재 확인 → 인증 코드 발송
3. 인증 코드 입력 (6자리, 유효시간 5분)
4. 새 비밀번호 입력 + 확인
5. 비밀번호 변경 완료 → 기존 모든 세션 종료
```

### 4.2 비즈니스 룰

| # | 규칙 |
|---|------|
| BR-PW-01 | 최근 3개 비밀번호와 동일한 비밀번호로 변경 불가 |
| BR-PW-02 | 비밀번호 재설정 링크/코드 발송은 1시간 내 최대 5회 |
| BR-PW-03 | 소셜 로그인 전용 계정은 비밀번호 찾기 불가 (안내 메시지 표시) |

## 5. 판매자 전환

### 5.1 프로세스

```
1. 구매자 계정으로 로그인
2. 판매자 전환 신청
3. 추가 정보 입력
   ├── 실명 인증 (본인 확인)
   ├── 연락처 인증
   ├── 전문 분야 선택 (1개 이상)
   ├── 정산 계좌 등록
   └── [선택] 사업자 등록증 업로드
4. 관리자 승인 (자동/수동)
5. 판매자 기능 활성화
```

### 5.2 판매자 유형

| 유형 | 조건 | 혜택 |
|------|------|------|
| **개인 판매자** | 실명 인증 + 계좌 등록 | 3.3% 원천징수 |
| **사업자 판매자** | 사업자등록증 인증 + 계좌 등록 | 세금계산서 발행 가능 |

## 6. 예외 처리

| 시나리오 | 처리 방안 |
|----------|-----------|
| 이메일 인증 메일 미수신 | 재발송 버튼 (60초 쿨다운), 스팸함 확인 안내 |
| 소셜 로그인 서비스 장애 | "일시적으로 이용이 어렵습니다" 안내 + 대체 로그인 수단 표시 |
| 가입 도중 이탈 | 임시 저장 후 24시간 내 이어가기 가능 |
| 비활성 계정 | 1년 미접속 시 휴면 처리 → 로그인 시 본인 인증 후 재활성화 |
| 계정 잠금 해제 요청 | 고객센터 1:1 문의 또는 이메일 인증으로 해제 |
