# 개인정보처리방침

> 본 문서는 프리랜서 마켓플레이스 플랫폼의 개인정보처리방침 기획안입니다.
> 실제 서비스 운영 시 법률 전문가의 검토 및 개인정보보호위원회 기준을 반영하여 최종 확정해야 합니다.

---

## 1. 수집 항목

### 1.1 필수 수집 정보

| 수집 시점 | 수집 항목 | 수집 근거 |
|-----------|-----------|-----------|
| **이메일 가입** | 이메일, 비밀번호, 닉네임 | 서비스 이용 계약 이행 |
| **소셜 로그인** | 소셜 계정 ID, 이메일, 프로필 이미지 (Provider 제공 범위) | 서비스 이용 계약 이행 |
| **판매자 전환** | 실명, 연락처 (휴대폰번호), 정산 계좌 (은행명/계좌번호/예금주) | 거래 및 정산 이행 |
| **개인 판매자** | 주민등록번호 (원천징수 목적), 사업소득 신고용 정보 | 세법 의무 이행 |
| **사업자 판매자** | 사업자등록번호, 대표자명, 사업장 주소 | 세금계산서 발행 |
| **결제** | 결제 수단 정보 (PG사 토큰), 결제 금액, 결제 일시 | 결제 처리 |
| **본인 인증** | 이름, 생년월일, 성별, 내/외국인, 휴대폰번호, CI/DI | 본인 확인 |

### 1.2 선택 수집 정보

| 수집 시점 | 수집 항목 | 수집 근거 |
|-----------|-----------|-----------|
| **프로필 설정** | 프로필 이미지, 소개문, 경력 사항, 포트폴리오 | 서비스 품질 향상 |

| **고객 문의** | 문의 내용, 첨부파일 | 고객 상담 |

|-----------|-----------|------|

## 2. 이용 목적

### 2.1 수집 정보별 이용 목적 매핑

| 수집 항목 | 이용 목적 |
|-----------|-----------|
| **이메일** | 회원 식별, 로그인, 알림 발송, 비밀번호 재설정 |
| **비밀번호 (해시)** | 본인 인증 |
| **닉네임** | 서비스 내 표시명, 커뮤니케이션 |
| **프로필 이미지** | 서비스 내 프로필 표시 |
| **휴대폰번호** | 본인 인증, SMS/알림톡 발송, 긴급 연락 |
| **정산 계좌** | 판매 수익 정산 지급 |
| **주민등록번호** | 원천징수 신고 (개인 판매자) |
| **사업자등록번호** | 세금계산서 발행 (사업자 판매자) |
| **결제 정보** | 결제 처리, 환불 처리 |
| **접속 로그** | 부정 이용 방지, 보안 감사, 법적 의무 준수 |
| **검색 기록 (익명)** | 검색 품질 개선, 서비스 추천 |
| **기기/브라우저 정보** | 서비스 최적화, 보안 (이상 접속 감지) |

### 2.2 목적 외 이용 금지

회사는 수집된 개인정보를 위 명시된 목적 이외의 용도로 이용하지 않으며, 이용 목적이 변경될 경우 사전에 동의를 받습니다.

| 권리 | 행사 방법 | 처리 기한 | 비고 |

| **동의 철회** | 마이페이지 설정 또는 고객센터 요청 | 즉시 | 마케팅 동의 등 선택 동의 |

### 6.2 행사 방법

| 방법 | 상세 |
|------|------|
| **온라인** | 마이페이지 > 개인정보 관리 > 열람/정정/삭제/동의 철회 |
| **고객센터** | 이메일 (privacy@example.com), 전화 (02-XXXX-XXXX) |
| **서면** | 개인정보 열람/정정/삭제 요청서 작성 후 우편 발송 |

### 6.3 처리 절차

```
요청 접수
    │
    ▼
본인 확인 (로그인 또는 신분증 확인)
    │
    ▼
요청 내용 검토
    │
    ├── 즉시 처리 가능 → 처리 후 결과 통지
    │
    └── 거부/일부 제한 사유 존재
        │
        ├── 법정 보관 의무 항목 (거래 기록 등)
        ├── 다른 사람의 생명/신체에 위해
        └── 기타 법률에 의한 제한
             │
             └── 사유 설명 후 부분 처리
```

### 6.4 법정대리인 권리

만 14세 미만 아동의 개인정보는 수집하지 않습니다 (BR-AUTH-05). 만약 수집 사실이 확인되면 해당 정보를 즉시 삭제합니다.

## 7. 보호 조치

### 7.1 기술적 보호 조치

| 조치 | 내용 |
|------|------|
| **전송 구간 암호화** | TLS 1.3 전 구간 적용 (HTTPS 강제) |
| **저장 데이터 암호화** | AES-256 암호화 (계좌번호, 주민등록번호 등 민감 데이터) |
| **암호화 키 관리** | AWS KMS를 통한 키 관리 |
| **비밀번호 해싱** | bcrypt (cost factor 12) 단방향 해싱 |
| **접근 통제** | IP 기반 접근 제어, VPN 필수 (관리자) |
| **세션 관리** | JWT (Access 15분 / Refresh 7일), HttpOnly Cookie |
| **입력값 검증** | Prisma ORM 바인딩, XSS 필터링, CSRF 토큰 |
| **Rate Limiting** | IP 기반 100req/min, 사용자 기반 300req/min |
| **파일 검증** | MIME 타입 검증, 크기 제한 (50MB), 바이러스 스캔 |
| **취약점 점검** | npm audit 자동화, 월 1회 보안 스캔 (OWASP ZAP) |

### 7.2 관리적 보호 조치

| 조치 | 내용 |
|------|------|
| **접근 권한 관리** | 최소 권한 원칙, 역할 기반 접근 통제 (RBAC) |
| **개인정보 취급자 교육** | 연 2회 개인정보 보호 교육 실시 |
| **내부 관리 계획** | 개인정보 내부 관리 계획 수립 및 시행 |
| **비밀유지 의무** | 개인정보 취급자 비밀유지 서약 |
| **감사 로그** | 관리자 모든 활동 기록 (BR-ADM-02), 2년 보관 |
| **접근 로그** | 개인정보 조회/수정/삭제 로그 기록, 1년 보관 |
| **2단계 인증** | 관리자 계정 2FA 필수 (BR-ADM-01) |
| **비밀번호 정책** | 관리자 90일마다 변경 (BR-ADM-05), 최근 3개 재사용 불가 (BR-PW-01) |

### 7.3 물리적 보호 조치

| 조치 | 내용 |
|------|------|
